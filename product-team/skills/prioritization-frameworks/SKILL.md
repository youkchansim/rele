# Prioritization Frameworks

Multi-framework prioritization guide for product decisions. Select the right framework based on context.

## Framework Selection Guide

| Situation | Recommended Framework | Why |
|-----------|----------------------|-----|
| Quick prioritization, solo dev | ICE | Simple 3-factor scoring |
| Data-rich environment, team alignment | RICE | Explicit reach factor for scaling |
| User satisfaction analysis | Kano Model | Categorizes feature types by user delight |
| Outcome-driven innovation | Opportunity Score | Identifies underserved user needs |
| Stakeholder alignment, release planning | MoSCoW | Clear must/should/could/won't buckets |
| Complex multi-criteria decisions | Weighted Scoring | Customizable criteria weights |

## ICE / RICE

Impact x Confidence x Ease scoring. RICE adds Reach for scale context.
See `references/ice-rice.md` for calculation details, examples, and calibration tips.

## Opportunity Score / Kano Model

Outcome-driven and satisfaction-based prioritization for user-centric decisions.
See `references/opportunity-kano.md` for ODI formula, Kano categories, and survey templates.

## MoSCoW / Weighted Scoring

Categorical and multi-criteria prioritization for stakeholder alignment.
See `references/moscow-weighted.md` for bucket rules, scoring matrices, and facilitation guides.
