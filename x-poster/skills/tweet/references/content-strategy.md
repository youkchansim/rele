# Content Strategy Guide — Scheduler App X Marketing

## App Positioning

Scheduler is an AI-powered scheduling app for ADHD users. It leverages Apple Intelligence and HealthKit to recommend optimal schedules based on personal energy patterns.

**Core keywords:** ADHD, productivity, AI scheduling, Apple Intelligence, energy management, indie dev, Build in Public

**Target audience (global):**
- ADHD/neurodivergent community
- Productivity tool enthusiasts
- iOS/Apple ecosystem users
- Indie developer community

---

## Commit → Content Mapping Rules

Analyze commit message patterns to auto-select the appropriate content type.

**IMPORTANT: Prioritize user-facing changes.** Internal refactoring, code cleanup, and invisible bug fixes should be deprioritized or skipped entirely.

### 1. New Feature (feat, implement, add)
**Content type:** Product showcase
**Tone:** Excited, proud
**Structure:** Feature description → Why it was built → User value
**Hook examples:** "Just shipped:", "New feature drop:", "You asked, I built:"

### 2. Bug Fix (fix, resolve)
**Content type:** Build in Public (struggle story)
**Tone:** Honest, humorous
**Structure:** Problem found → Debugging journey → Solution → Lesson
**Hook examples:** "Spent 3 hours debugging", "The bug that humbled me", "Sometimes the fix is one line"

### 3. Performance Improvement (improve, optimize, refactor)
**Content type:** Tech decision sharing
**Tone:** Educational, analytical
**Structure:** Problem → Options → Why this choice → Result
**Hook examples:** "Why I refactored X to Y", "Performance tip:", "Before vs After:"

### 4. UI/UX Change (UI, design, View)
**Content type:** Before/after showcase
**Tone:** Visual, comparison-focused
**Structure:** Before → After → Why the change
**Hook examples:** "Small design change, huge impact", "UI glow-up:", "Details matter:"

### 5. AI-Related (AI, parsing, prompt, LLM)
**Content type:** Apple Intelligence insight
**Tone:** Technical yet accessible
**Structure:** Technical challenge → Solution → User experience impact
**Hook examples:** "On-device AI tip:", "What I learned building with Apple Intelligence:", "AI ≠ magic, here's what it actually takes:"

### 6. HealthKit-Related (HealthKit, health, energy, sleep)
**Content type:** ADHD/energy insight
**Tone:** Empathetic, insight-driven
**Structure:** ADHD observation → Data validation → How the app helps
**Hook examples:** "Your body knows your schedule", "ADHD + data =", "Energy tracking changed everything:"

### 7. Multiple Commits (summary)
**Content type:** Build in Public dev log
**Tone:** Summary, transparent
**Structure:** What was done → Numbers → Next steps
**Hook examples:** "Week X building in public:", "This week's indie dev log:", "7 days of shipping:"

## Content Category Details

### BIP (Build in Public)
- Transparent sharing of the dev process
- Include both failures and wins
- Emphasize numbers and concrete facts
- Build community connection

### Showcase (Product Showcase)
- Focus on one specific feature
- Emphasize user-perspective value
- Mention screenshots/video when possible
- Avoid direct CTA (attract interest only)

### Insight (ADHD/Productivity Insight)
- Based on personal experience
- Backed by data
- Evoke ADHD community empathy
- Focus on insight, not the app itself

### Tech (Tech Decision Sharing)
- Developer audience
- Specific rationale for tech choices
- Code snippets or architecture decisions
- Provide educational value

## X Algorithm Optimization Rules

### Engagement Weights
- Reply: ×13.5
- RT: ×20
- Profile click: ×12
- Bookmark: ×10
- Author reply: ×75

### Optimization Strategy
1. **Length:** 71-100 chars optimal (or 4-8 tweet thread)
2. **Reply bait:** Always end with a question
3. **Hashtags:** 1-2 max (3+ = spam)
4. **External links:** Minimize (algorithm suppresses)
5. **Media:** Images ×2-3, video ×10 engagement
6. **First tweet:** Shocking number or bold claim

### Posting Time (KST)
- **Optimal:** Wednesday 11 PM (US East morning, EU afternoon)
- **Secondary:** Tue/Thu 10 PM–midnight

### Don'ts
- Excessive external links
- 3+ hashtags
- "Buy my app" hard-sell tone
- Copying/modifying other tweets
